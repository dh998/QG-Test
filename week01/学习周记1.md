# 宿舍管理系统业务逻辑
## 1. 实体类
User.java / RepairOrder.java

## 2. Mybatis映射层（mapper接口+xml文件）
### 1.  Mapper接口（src/main/java/com/dorm/repair/mapper/）
     UserMapper
定义增删改查方法，MyBatis 通过 xml 文件绑定 SQL，方法名和 xml 中 id 严格一致                   
RepairOrderMapper
创建报修单 ，根据学生账号查看报修记录，取消报修单，修改报修单状态，删除报修单，查询所有报修单（根据订单号查看报修单详情）
### 2. Mapper映射文件(src/main/resources/mapper/)
和接口一一对应 实现SQL语句 结果映射处理数据库下划线字段->驼峰字段 以防无法读取数据的情况发生
UserMapper.xml / RepairMapper.xml
## 3. 工具类（util包）
MybatisUtil.java（仿照之前黑马的MyBatisDemo）
获取SqlSession，封装Mybatis核心操作  
## 4. 业务逻辑 （所有功能区的汇总）
### 1. UserService.java（编写用户业务：注册/登录/改密码/绑定宿舍/查看信息等）
### 2. RepairOrderService.java（报修单业务：创建 / 查询 / 取消 / 改状态 / 删除）增删改查
## 5. 终端交互(Main.java)
登录界面 (主菜单)/ 用户注册 / 学生登录 /学生菜单（首次绑定宿舍）/管理员菜单
